Model Extension Guide

This document explains how to add a new deep learning model to the platform. The model system uses a registry pattern — each model is self-contained in its own file, and adding a new one requires no changes to existing model code.

Architecture Overview

utils/
  models/                  # Package — all model implementations
    __init__.py            # Registry: MODEL_REGISTRY dict
    base.py                # BaseModel (abstract base class)
    mlp.py                 # MLPModel
    cnn.py                 # CNN1DModel
    rnn.py                 # RNNModel
    lstm.py                # LSTMModel
    gru.py                 # GRUModel
    transformer.py         # TransformerTabularModel
    your_new_model.py      # ← Your new model goes here
  model_utils.py           # Training loop, predict, evaluate (no model classes)
  plot_utils.py            # Visualization helpers
  data_utils.py            # Data loading, cleaning

templates/
  index.html               # Frontend — add dropdown option here

static/js/
  app.js                   # Frontend — update params/toggleModelParams here

main.py                    # Backend — param building uses registry
Key principle: The MODEL_REGISTRY in utils/models/__init__.py is the single source of truth. Every model class is registered there with its params schema. The backend (main.py) reads the schema to build parameters automatically — you don't need to edit main.py for most new models.

Quick-Start: 3-Step Checklist

  1. Create the model file → utils/models/your_model.py
    Inherit from BaseModel, implement __init__ and forward
  2. Register in the registry → utils/models/__init__.py
    Import + add entry to MODEL_REGISTRY with params schema
  3. Add frontend UI → templates/index.html + static/js/app.js
    Dropdown option + parameter controls

Step 1: Create the Model Class

Create a new file in utils/models/. Your class must:

  • Inherit from BaseModel (which inherits from nn.Module)
  • Accept (self, input_dim, output_dim, **kwargs)
  • Extract your parameters from kwargs with sensible defaults
  • Implement forward(self, x)

Example: utils/models/attention_rnn.py

"""Attention RNN — example of adding a custom model."""

from torch import nn
from .base import BaseModel


class AttentionRNNModel(BaseModel):
    """RNN with attention mechanism."""

    def __init__(self, input_dim, output_dim, **kwargs):
        super().__init__(input_dim, output_dim, **kwargs)
        # Extract YOUR params from kwargs (fallback to defaults)
        hidden_size = kwargs.get("hidden_size", 64)
        num_layers = kwargs.get("num_layers", 2)
        dropout = kwargs.get("dropout", 0.2)

        # Build your architecture
        self.rnn = nn.LSTM(
            input_size=1, hidden_size=hidden_size,
            num_layers=num_layers, batch_first=True,
            dropout=dropout if num_layers > 1 else 0,
        )
        self.attention = nn.Linear(hidden_size, 1)
        self.fc = nn.Sequential(
            nn.Dropout(dropout),
            nn.Linear(hidden_size, output_dim),
        )

    def forward(self, x):
        x = x.view(x.size(0), -1, 1)
        out, _ = self.rnn(x)                # (batch, seq, hidden)
        attn_weights = self.attention(out)   # (batch, seq, 1)
        attn_weights = attn_weights.softmax(dim=1)
        context = (out * attn_weights).sum(dim=1)
        return self.fc(context)

Step 2: Register the Model

Open utils/models/__init__.py and make two changes:

2a. Import your class

from .base import BaseModel
from .mlp import MLPModel
from .cnn import CNN1DModel
...
from .attention_rnn import AttentionRNNModel   # ← Add this

2b. Add entry to MODEL_REGISTRY

Each entry has:

FieldDescription
classThe model class
nameDisplay name (appears in the frontend dropdown)
paramsDict of hyperparameters. Each param needs type, default, label
MODEL_REGISTRY = {
    ...
    "transformer": { ... },
    "attention_rnn": {
        "class": AttentionRNNModel,
        "name": "Attention RNN",
        "params": {
            "hidden_size": {"type": "int", "default": 64, "label": "Hidden size"},
            "num_layers": {"type": "int", "default": 2, "label": "Number of layers"},
            "dropout": {"type": "float", "default": 0.2, "label": "Dropout"},
        },
    },
}

Param type reference

TypeBackend conversionExample
"int"int(raw)hidden_size, num_layers
"float"float(raw)dropout, learning_rate
"bool"raw in (True, "true", 1)bidirectional
"string"passed as-ishidden_layers (comma-separated)
That's all for the backend! The create_model() and main.py param builder automatically read from the registry — no need to modify model_utils.py or main.py unless your model needs special handling.

Step 3: Add Frontend UI

3a. Dropdown option — templates/index.html

Find the <select id="modelType"> element and add your model:

<select id="modelType" onchange="toggleModelParams()">
  <option value="mlp">MLP (Fully Connected)</option>
  ...
  <option value="transformer">Transformer (Encoder)</option>
  <option value="attention_rnn">Attention RNN</option>  <!-- ← Add -->
</select>

3b. Parameter UI — templates/index.html

After the transformerParams div, add a div for your model's parameters:

<div id="attentionRnnParams" style="display:none;">
  <div class="form-row">
    <div class="input-group">
      <label>Hidden size</label>
      <input type="number" id="attnHiddenSize" value="64" min="8" max="512">
    </div>
    <div class="input-group">
      <label>Number of layers</label>
      <input type="number" id="attnNumLayers" value="2" min="1" max="10">
    </div>
  </div>
</div>

3c. Toggle visibility — static/js/app.js

Update the toggleModelParams() function:

function toggleModelParams() {
  const type = document.getElementById("modelType").value;
  document.getElementById("mlpParams").style.display = type === "mlp" ? "block" : "none";
  document.getElementById("cnnParams").style.display = type === "cnn" ? "block" : "none";
  const seq = ["rnn", "lstm", "gru"];
  document.getElementById("seqParams").style.display = seq.includes(type) ? "block" : "none";
  document.getElementById("transformerParams").style.display = type === "transformer" ? "block" : "none";
  document.getElementById("attentionRnnParams").style.display = type === "attention_rnn" ? "block" : "none";
}

3d. Send params — static/js/app.js

In the startTraining() function, add your model's params to the request body:

const params = {
  target_col: targetCol,
  model_type: document.getElementById("modelType").value,
  ...
  d_model: parseInt(document.getElementById("transDModel").value) || 64,
  hidden_size: parseInt(document.getElementById("attnHiddenSize").value) || 64,
  num_layers: parseInt(document.getElementById("attnNumLayers").value) || 2,
  ...
};

Example: Minimal Model Template

Use this template when creating a new model file:

"""Short description of your model."""

from torch import nn
from .base import BaseModel


class YourModelName(BaseModel):
    """One-line description."""

    def __init__(self, input_dim, output_dim, **kwargs):
        super().__init__(input_dim, output_dim, **kwargs)
        # 1. Extract your params from kwargs
        my_param = kwargs.get("my_param", default_value)
        # 2. Build layers
        # 3. Store them as self.xxx = nn.Sequential(...)

    def forward(self, x):
        # 1. x shape: (batch, input_dim)
        # 2. Apply your layers
        # 3. Return (batch, output_dim) for regression
        #    or (batch, n_classes) for classification
        return x
